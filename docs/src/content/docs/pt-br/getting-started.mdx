## Criando um novo projeto

Starstruck é um tema de documentação cheio de funcionalidades construído em cima do framework [Astro](https://astro.build).

Você pode criar um novo projeto Astro + Starstruck usando o seguinte comando:

<Tabs>
<TabItem label="npm">

```sh
# cria um novo projeto com npm
npm create astro@latest -- --template starstruck
```

</TabItem>
<TabItem label="pnpm">

```sh
# cria um novo projeto com pnpm
pnpm create astro --template starstruck
```

</TabItem>
<TabItem label="Yarn">

```sh
# cria um novo projeto com yarn
yarn create astro --template starstruck
```

</TabItem>
</Tabs>

Isso irá criar um novo [diretório de projeto](/pt-br/guides/project-structure/) com todos os arquivos e configurações necessárias para o seu site.

:::tip[Veja-o em ação]
Experimente Starstruck no seu navegador:
[abrir o template no StackBlitz](https://stackblitz.com/github/withastro/starstruck/tree/main/examples/basics).
:::

## Criando conteúdo com Starstruck

Starstruck está pronto para você adicionar novo conteúdo, ou trazer seus arquivos existentes!

### Formatos de arquivo

Starstruck suporta escrever conteúdo em Markdown e MDX. (Você pode adicionar suporte para Markdoc instalando a [integração experimental Astro Markdoc](https://docs.astro.build/en/guides/integrations-guide/markdoc/).)

### Adicione páginas

Adicione novas páginas ao seu site automaticamente ao criar arquivos `.md` ou `.mdx` em `src/content/docs/`. Adicione subdiretórios para organizar seus arquivos, e para criar múltiplos segmentos de caminho:

```
src/content/docs/ola-mundo.md => seu-site.com/ola-mundo
src/content/docs/guias/faq.md => seu-site.com/guias/faq
```

### Frontmatter com segurança de tipos

Todas as páginas do Starstruck compartilham um [conjunto comum de propriedades frontmatter](/pt-br/reference/frontmatter/) para controlar como a página aparece:

```md
---
title: Olá, Mundo!
description: Essa é uma página no meu site com Starstruck
---
```

Se você esquecer de algo importante, Starstruck vai te informar.

## Fazendo deploy do seu website Starstruck

Assim que você tiver criado e customizado seu website Starstruck, você pode fazer deploy dele para um servidor web ou plataforma de hospedagem de sua escolha, incluindo Netlify, Vercel, GitHub Pages e vários outros.

[Aprenda sobre como fazer deploy de um site Astro na documentação do Astro.](https://docs.astro.build/pt-br/guides/deploy/)

## Atualizando Starstruck

:::tip
Como o Starstruck é um software em beta, haverá frequentes atualizações e melhorias. Certifique-se de atualizar Starstruck constantemente!
:::

Starstruck é uma integração Astro, e é atualizado como qualquer outra integração `@astrojs/*`:

<Tabs>
<TabItem label="npm">

```sh
# atualiza Starstruck com npm
npm install @awe-player/starstruck@latest
```

</TabItem>
<TabItem label="pnpm">

```sh
# atualiza Starstruck com pnpm
pnpm upgrade @awe-player/starstruck --latest
```

</TabItem>
<TabItem label="Yarn">

```sh
# atualiza Starstruck com yarn
yarn upgrade @awe-player/starstruck --latest
```

</TabItem>
</Tabs>

Você pode ver uma lista completa das mudanças feitas para cada versão no [histórico de mudanças do Starstruck](https://github.com/withastro/starstruck/blob/main/packages/starstruck/CHANGELOG.md).
